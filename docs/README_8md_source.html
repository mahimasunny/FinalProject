6 Introduction
7 ===
8 User Datagram Protocol(UDP) is used in internet applications to implement low-latency and loss-tolerating commenction. UDP enalbles a process to process communication. This project aims to implement a bidirectional UDP message transferring mechanism between a UDP client and a UDP Server.

11 Project Details
12 ===
13 This project is a UDP based iOT application that comprises of a Thermostat(UDP Server) and a Room Heater (UDP client). The thermostat and the heater communicate to each other via UDP. There is also an internal manager in heater side that controls the heater, that is turn the heater on or off.
14  The Thermostat can send the current temperature value to the heater in JSON format. The Thermostat listens to a predefined port for the messages from the Heater.
15 
16  Heater is a finite state machine that has an on state and an off state. It is initially set to an off state.
17 The internal Manager sends a message to turn on the heater. If the heater is on, it requests for a temperature value from the thermostat. Thermostat then sends back the requested temperature value.

43 *Steps in implementing UDP Server and Client with sockets*
44 ---
45 The following are the steps I followed for the completion of this project.
46 1. Socket creation
47  ```
48  #include <sys/socket.h>
49 
50  ...
51 
52  if ((fd = socket(AF_INET, SOCK_DGRAM, 0)) < 0) {
53  perror("cannot create socket");
54  return 0;
55  }
56 
57  ```
58  where, sockfd is the socket descripton, domain refers to communication domain for example AF_INET (IPv4 protocol) , AF_INET6 (IPv6 protocol), type is SOCK_DGRAM for UDP and protocol value is the value for Internet Protocol which is 0.
59  Here all the input parameters are integers. socket mehod is defined in sys/socket.h.
60 
61 2. Identify a socket
62  Sockets are assigned port numbers or transport addresses.
63 
64  In server side
65  ```
66  struct sockaddr_in myaddr;
67 
68  /* bind to an arbitrary return address */
69  /* because this is the client side, we don't care about the address */
70  /* since no application will initiate communication here - it will */
71  /* just send responses */
72  /* INADDR_ANY is the IP address and 0 is the socket */
73  /* htonl converts a long integer (e.g. address) to a network representation */
74  /* htons converts a short integer (e.g. port) to a network representation */
75 
76  memset((char *)&myaddr, 0, sizeof(myaddr));
77  myaddr.sin_family = AF_INET;
78  myaddr.sin_addr.s_addr = htonl(INADDR_ANY);
79  myaddr.sin_port = htons(PORT);
80 
81  if (bind(fd, (struct sockaddr *)&myaddr, sizeof(myaddr)) < 0) {
82  perror("bind failed");
83  return 0;
84  }
85  ```
86  In the server side, Bind method forcefully binds the PORT address.
87 
88 
89  Client side: send message to the PORT to which binding was done at the server side.
90  ```
91  memset((char *)&myaddr, 0, sizeof(myaddr));
92  myaddr.sin_family = AF_INET;
93  myaddr.sin_addr.s_addr = htonl(INADDR_ANY);
94  myaddr.sin_port = htons(0);
95  ```
96 
97 3. Send message to a server from a client
98  a. Client side
99  sendto is used to send a message from client to server
100 
101  ```
102  sendto(fd, buf, strlen(buf), 0, (struct sockaddr *)&remaddr, slen);
103  ```
104  fd refers to the name of the socket.
105  my_message, provides the starting address of the message we want to send.
106  The flags parameter is 0 for UDP sockets.
107  The remaddr defines the destination address and port number for the message. socklen_t is the length of the address structure: sizeof(struct sockaddr_in).
108 
109  b. Receive message at server
110  ```
111  /* now loop, receiving data and printing what we received */
112  for (;;) {
113  printf("waiting on port %d\n", PORT);
114  recvlen = recvfrom(fd, buf, BUFSIZE, 0, (struct sockaddr *)&remaddr, &addrlen);
115  printf("received %d bytes\n", recvlen);
116  if (recvlen > 0) {
117  buf[recvlen] = 0;
118  printf("received message: \"%s\"\n", buf);
119  }
120  }
121  /* never exits */
122 
123  ```
124  fd refers to the name of the socket.
125  buf is the input data and recvlen is the length of that data
126  The flags parameter is 0 for UDP sockets.
127  remaddr is used to recognize the sender.
128 
129 
130 4. On the client, send a message
131 
132  Message from client to server
133  ```
134  recvlen = recvfrom(s, buf, BUFSIZE, 0, (struct sockaddr *)&remaddr, &addrlen);
135  ```
136 
137  Reply from server to client send to remaddr
138  ```
139  sendto(s, buf, strlen(buf), 0, (struct sockaddr *)&remaddr, addrlen)
140 
141  ```
142 
143  At this stage the client can also send back a message to the server
144 
145 
146 5. Close the socket
147  Eventhough there is no communication to be terminated in UDP, the system needs to be closed to free up memory.
148  ```
149  close(fd);
150 
151  ```
152  fd refers to the file descriptor
